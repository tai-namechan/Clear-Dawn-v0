# ADR-0002: user_id スコープ・シングルユーザードメインとする

- 状態: 承認済み

## 文脈

Clear Dawn は個人のセルフマネジメントアプリであり、組織・店舗・チームの概念を持たない。
一方で複数ユーザーが同一環境に登録できるため、データの混入防止が必須となる。

## 決定

- 全ユーザーデータは `user_id` カラムで所有者を限定する
- 取得クエリの WHERE に必ず `user_id` 条件を含め、Policy で認可を二重に守る
- `Admin` / `Shop` / `tenant_id` などマルチテナント・組織前提の設計は持ち込まない
- 例外はグローバルマスタ（`matrix_rows` / `metrics` 等、ユーザー入力を含まないテーブル）のみ

## 理由

- 全テーブルスキャンとデータ混入を防ぐ（インデックスも user_id 先頭で設計できる）
- Policy（403）とクエリスコープ（404 / 空）の二重防御で権限外アクセスを塞ぐ
- ドメインを個人に固定することで設計・実装・テストが単純になる

## 影響

- Feature テストでは「他ユーザーのデータが返らない・操作できない」ことを必ずアサートする
- Export API も認証ユーザーのデータのみを出力する
