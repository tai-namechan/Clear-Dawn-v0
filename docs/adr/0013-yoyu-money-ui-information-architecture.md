# ADR-0013: ヨユウ「お金の余裕」UI 情報設計再編

- ステータス: 承認（実装反映）
- 日付: 2026-07-21
- 関連: [ADR-0010](./0010-yoyu-money-margin-domain.md)、[yoyu-money-margin-design.md](../product/yoyu-money-margin-design.md)

## コンテキスト

Money 機能はドメイン・計算は揃っていたが、画面が機能別タブ（11項目）に並び、ユーザーが「今月あといくら使えるか」「次に何を調整するか」を判断しづらかった。

## 決定

1. 最上位ナビを **ホーム / 今月 / 資産・返済 / 明細 / 計画** の5分類へ再編する
2. 設定は歯車アイコンで独立させる
3. 既存URLは維持し、対応するナビ・内部タブを選択状態にする
4. ホームは「安全に使える金額」を最優先表示し、残高タイムライン・調整候補へ誘導する
5. 初回セットアップ進捗は既存データから導出し、完了フラグ用マイグレーションは追加しない
6. カード枠は信用枠情報として分離し、資産サマリーへ加算しない
7. シミュレーション保存は実データを変更しない（apply のみ反映）

## 結果

- 共通シェル `MoneyPageShell` とセクションタブで全Money画面を統一
- デスクトップ本文幅を拡大（〜1280–1360px）、モバイルは下部ナビとカード一覧
- 計算式・不変条件（ADR-0010）は変更しない

## アクセシビリティ方針

- focus ring を消さない
- ナビ・ドロワーに aria 属性を付与
- 色だけで状態を伝えない（バッジ文言を併用）
- 主要操作は十分なクリック領域を確保
