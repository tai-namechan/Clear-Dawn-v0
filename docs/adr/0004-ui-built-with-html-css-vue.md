# ADR-0004: UI は画像一枚に頼らず HTML/CSS/Vue で構成する

- 状態: 承認済み（UI 仕様書 v2 で決定）

## 文脈

Clear Dawn の世界観（夜明け前の深紫のサイドバー、大理石の背景、月・星・植物線画）は
AI 生成画像で表現できる品質を目指すが、画像一枚で画面を作ると
レスポンシブ・テキスト編集・状態管理・ホバー/クリック・メニュー追加に対応できない。

## 決定

- UI として動くもの（サイドバー全体・テーブル・ボタン・チェックボックス・カード・
  モーダル・文字入り UI）は必ず HTML/CSS/Vue コンポーネントで実装する
- 画像にしてよいのは世界観素材（大理石背景・月・星・植物線画・装飾ライン）のみ
- サイドバーは CSS グラデーション + SVG 装飾の重ね合わせで構成する
- アイコンは SVG（Lucide）で統一し、絵文字・フォントアイコンは使わない

## 理由

- レスポンシブ・状態管理・インタラクションに耐える（実運用アプリとして必須）
- デザイントークン（`resources/css/app.css`）で色・質感を一元管理できる
- `currentColor` / CSS 変数により、装飾の色調整を CSS だけで完結できる

## 影響

- デザイン変更は原則トークンと CSS の修正で行い、画像の再生成に依存しない
- 素材の扱いは [../design/assets.md](../design/assets.md) のルールに従う
- 実装レビューでは [../design/ui-quality-checklist.md](../design/ui-quality-checklist.md) を通過させる
