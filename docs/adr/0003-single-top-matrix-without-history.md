# ADR-0003: TOP Matrix はユーザーごと 1 つ・履歴を持たない

- 状態: 承認済み（Phase 1 仕様書 v0.3 で決定）

## 文脈

TOP Matrix を日別・週別・月別にスナップショット管理する案もあり得たが、
Clear Dawn における TOP の役割は「今の自分の状態の見える化」であり、
過去の状態の閲覧ではない。

## 決定

- TOP Matrix はユーザーごとに 1 つだけ保持する。日別・週別・月別管理はしない
- TOP 自体は履歴を持たない
- 「何をしたか」の履歴は、メモ・日次/週次振り返り・実行履歴（activity_logs）側に残す

## 理由

- TOP は常に「現在」を映す 1 枚のシートであるべき（コンセプトとの一致）
- スナップショット管理はデータ量・UI 複雑度を大きく増やす割に、
  振り返りモジュールで代替できる
- 履歴の責務を L2（記録層）に寄せることで、各モジュールの責務が明確になる

## 影響

- セル・セル項目の変更は上書きであり、変更履歴テーブルは作らない
- 完了の事実を残す場合は activity_logs にイベントとして記録する（M1 から `matrix_item_completed` /
  `matrix_item_reopened`。実行履歴 UI は M3）
- 日次振り返りの「当日完了実績」は `completed_at` を用いたクエリ参照で実現する
