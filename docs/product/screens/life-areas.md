# 領域管理（Life Area）仕様

対象フェーズ: Phase 1（TOP Matrix と同時に提供）

## 目的

TOP Matrix の横軸である「領域（Life Area）」を定義・管理する。
例: 仕事、野球、バイオリン、プライベート。

領域は TOP Matrix の列であると同時に、メモ・ルーティン・記録・Finance など
L2 モジュール全体を貫く横串タグでもある。

## 画面

| 画面 | ルート | 概要 |
|---|---|---|
| 領域管理 | GET /life-areas | 領域の一覧・追加・編集・並び替え・非表示 |

導線はサイドバーに置かず、以下の 2 箇所とする。

1. TOP マトリクスのヘッダー編集ボタン
2. 設定 > 領域管理

## 主要操作

| 操作 | 入力 | 挙動 |
|---|---|---|
| 追加 | name, color | 領域を作成し、固定行ぶんのセルを利用可能にする |
| 名称・色変更 | name, color | 表示のみ変わる。既存セル・項目は保持 |
| 並び替え | sort_order | TOP Matrix の列順に即時反映 |
| 非表示化 | is_active = false | TOP Matrix から列が消える。セル・項目データは保持 |
| 再表示 | is_active = true | 列とデータが復帰する |

## 設計方針

- 領域の「削除」は原則行わず、`is_active = false`（非表示）を基本とする。
  物理削除・soft delete による列データ喪失を避けるため
- `deleted_at`（soft delete）はテーブル定義上持つが、ユーザー操作の既定は非表示化とする
- 領域に紐づく横串 FK（`life_area_id`）は非表示化しても保持する
- 想定件数は 1 ユーザーあたり数個〜10 個程度。一覧は全件取得で問題ない

## バリデーション（実装時の目安）

| 項目 | ルール |
|---|---|
| name | 必須、最大 50 文字、同一ユーザー内での重複は許容（警告のみ検討） |
| color | 必須、デザインシステムで定義するパレットから選択（自由 hex 入力は不可） |
| sort_order | 整数。並び替え操作でサーバー側が採番 |

色パレットの定義は [../../design/design-system.md](../../design/design-system.md) に従う。

## 認可

- LifeAreaPolicy で「自分の領域のみ」操作可能にする
- 一覧取得は `where user_id = auth id` を必ず含める
