# ルーティン / トレーニング / 実行履歴仕様

対象フェーズ: Phase 2

## 目的

- ルーティン: 習慣・トレーニングメニューを定義し、迷わず実行できる状態にする
- 実行履歴: 「やった事実」を時系列で一覧化し、継続を可視化する

## 画面

| 画面 | ルート | 概要 |
|---|---|---|
| ルーティン管理 | GET /routines | ルーティンの一覧・作成・編集 |
| ルーティン実行 | GET /routines/{routine}/run | 当日の実行支援（種目ごとのチェック / 数値入力） |
| 実行履歴 | GET /history | 実行ログの時系列一覧 |

## ルーティン管理の主要操作

| 操作 | 入力 | 挙動 |
|---|---|---|
| 作成 | name, life_area_id(nullable), 実施頻度 | ルーティンを作成 |
| 種目（ステップ）管理 | name, 目標値(nullable), sort_order | ルーティン内の種目を追加・並び替え |
| 参考動画添付 | video_id | 種目に動画を紐づけ（Phase 3.5 以降） |
| 無効化 | is_active = false | 一覧・実行対象から外す。ログは保持 |

実施頻度は v0 では「毎日 / 曜日指定」の 2 種類から開始する（それ以上の複雑な繰り返しは未対応）。

## ルーティン実行の流れ

1. 「開始」ボタン（btn-accent）で実行セッションを開始
2. 種目ごとにチェック、必要なら数値（回数・重量・時間など）を入力
3. 「完了」で実行ログを確定。実行履歴と記録（該当する場合）へ反映

- 途中中断は「未完了のまま保存」を許容する（完遂率も含めて事実を残す）
- 実行中の画面はシートの世界観を保ちつつ、進行状態を明確に示す

## 実行履歴

実行履歴は以下を統合した時系列ビューとする。

| ソース | 内容 |
|---|---|
| ルーティン実行ログ | いつ・どのルーティンを・どこまで実行したか |
| TOP Matrix 完了ログ | セル項目「今やるべきこと」の完了イベント |

- 絞り込み: 期間 / 領域 / 種別（ルーティン・マトリクス）
- **イベントログ基盤（activity_logs）は Phase 2 着手前に設計を確定する。**
  後付けすると過去履歴が欠損するため、Phase 1 実装時点から完了切替イベントを記録するかは
  ロードマップ上の判断ポイント（[../../roadmap.md](../../roadmap.md) 参照）

## TOP Matrix との接続

- ルーティンは独立管理とする
- 当日実施予定のルーティンを TOP に補助表示するかは **未決定**（M3 設計時に判断）

## 設計方針

- 想定件数: 実行ログは年間数千件級。履歴一覧はページネーション + 期間絞り込み必須
- ログは soft delete しない（事実の記録として不変にする。誤操作訂正は許容）
- 数値入力された実績（重量・回数など）は記録モジュール（筋力）と重複させず、
  実行ログ側を正とし、記録画面が実行ログを参照する構成を推奨（M4 設計時に確定）
