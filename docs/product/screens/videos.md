# 動画仕様

対象フェーズ: Phase 3.5

## 目的

フォーム確認等の短尺動画を管理し、ルーティン（トレーニング）の種目に添付して
実行時に参照できるようにする。

## 画面

| 画面 | ルート | 概要 |
|---|---|---|
| 動画ライブラリ | GET /videos | アップロード済み動画の一覧・再生・削除 |

ルーティン実行画面・種目編集画面からも参照・添付できる。

## 主要操作

| 操作 | 入力 | 挙動 |
|---|---|---|
| アップロード | mp4 ファイル, title, life_area_id(nullable) | Object Storage へ保存しメタデータを登録 |
| 再生 | - | 署名付き URL で認証済みユーザーのみ再生 |
| ルーティンへ添付 | routine_step_id | 種目に紐づけ |
| 削除 | - | メタデータ soft delete + ストレージ上の実体削除は非同期 |

## 暫定方針（Phase 1 仕様書の決定事項を踏襲）

| 項目 | 暫定方針 |
|---|---|
| 保存先 | Laravel Cloud Object Storage |
| ローカル開発 | 動画実装時に MinIO を検討（それまで不要） |
| 形式 | mp4 を基本 |
| サイズ上限 | 初期は 1 本 50MB 目安 |
| 尺 | 基本 30 秒、上限 60 秒 |
| サムネイル | 初期は未対応。後で Queue で自動生成 |
| URL | 公開 URL ではなく認証済みユーザー向け。署名付き URL を検討 |

上限値・ストレージ構成は動画実装時（M6）に確定する。

## 設計方針

- DB には storage key・duration・size 等のメタデータのみ保持し、実体は Object Storage に置く
- アップロードはサイズ・形式・尺をサーバー側で検証する
- 想定件数: 数百件級。一覧はページネーション + サムネイル（対応後）
- TOP Matrix とは直接つながない（ルーティン経由の間接接続のみ）

## データ（ドラフト）

テーブル定義は [../../data/tables.md](../../data/tables.md) の `videos` を参照。

## 認可

- 自分の動画のみ再生・操作可能（Policy + 署名付き URL）
