# Finance 仕様（superseded）

> **置き換え済み**: 本ドラフトは実装しない。
> ヨユウ「お金の余裕」へ統合する。詳細は [ADR-0010](../../adr/0010-yoyu-money-margin-domain.md) と
> [yoyu-money-margin-design.md](../yoyu-money-margin-design.md) を正とする。
> 認証後の旧 URL `/finance` は `/yoyu/money` へ redirect する。

対象フェーズ: Phase 3（廃止）

## 目的

収支を見える化し、生活の土台を数字で把握する。
v0 では簡易収支（収入・支出の記録と月次サマリ）から開始する。

## 画面

| 画面 | ルート | 概要 |
|---|---|---|
| Finance | GET /finance | 当月の収支一覧 + 入力 + 月次サマリ |
| 月指定表示 | GET /finance?month=YYYY-MM | 過去月の閲覧 |

## 主要操作

| 操作 | 入力 | 挙動 |
|---|---|---|
| 収支登録 | date, type(income/expense), amount, category_id, memo(nullable) | エントリを作成 |
| 編集・削除 | 同上 | 上書き / 削除 |
| カテゴリ管理 | name, type | ユーザー別カテゴリの追加・編集 |
| 月次サマリ | month | 収入合計・支出合計・差引・カテゴリ別内訳 |
| 推移グラフ | 期間 | 月次収支の推移（チャートライブラリは記録と共通・未決定） |

## 設計方針

- 金額は **int（円）** で保持する。小数・浮動小数は使わない
- type は Enum（income / expense）で管理する
- カテゴリはユーザー別マスタ（`user_id` スコープ）。初期カテゴリは登録時に seed する
- 領域（`life_area_id`）タグは任意（例: プライベート）。必須にしない
- 想定件数: 月数十件〜、年間数百〜千件級。一覧は月単位表示でページネーション不要、
  年間集計は集計クエリで取得する

## スコープ（v0）

| 区分 | 内容 |
|---|---|
| 含める | 収入・支出の記録、カテゴリ、月次サマリ、推移グラフ |
| 含めない（未決定） | 予算管理、資産管理、口座連携、定期取引の自動登録 |

予算・資産管理まで広げるかは M5 着手前に判断する（[../../roadmap.md](../../roadmap.md) 未決定事項参照）。
