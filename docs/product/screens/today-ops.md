# 「今日/作戦」画面 仕様

対象: セルフマネジメントOS拡張 Phase 2〜4（2026-07-16 確定）
関連: [programs.md](./programs.md)、[routines.md](./routines.md)、[ADR-0011](../../adr/0011-five-data-kinds-and-rule-engine.md)

## 目的

既存 `/today`（今日やる）を、パフォーマンス管理領域の入口「今日/作戦」画面へ拡張する。
30〜60秒の状態入力から、今日のプログラム・作戦カード・食事の次の一手までを1画面で示す。
**ダッシュボード（TOP Matrix）とは別物であり、変更しない。**

## 表示順（上から · 2026-07-18 作戦優先）

1. 今日の日付と予定プログラム（現在週 / 今日のDAY）・選択日の選択肢
2. **今日の作戦カード（最優先・1件を大きく）**: 方針 / 理由 / 狙い / 確信度 / 選択肢 [実行・軽くやる・調整・見送り]
3. チェックイン状態バー（済なら薄く表示。未入力 or 「変更」で展開）
4. 追加の作戦カード（あれば折りたたみ級）
5. 今日のセッション概要 + 食事の残り目安（二次・2カラム）
6. 未入力・古い測定データ（後回し）
7. 受診依頼リスト（H7ロック中のみ · 後回し）
8. コンディション / 食事へのリンク

## ルール

- 作戦カードは決定論エンジン（Phase 4）が生成。承認されるまで今日のプランは変わらない。
- 承認は3段のうち A（今日のみ）をこの画面で完結。B/C はプログラム画面へ誘導。
- 割り込み提示はハードゲート（H1〜H7）のみ・1日1件まで。他はカード表示。
- 較正期間（データ蓄積4週未満）は警告系カードを出さず「較正中」と表示。

## 日付・タイムゾーン

- **日付の正はユーザー TZ**（`users.timezone` → `UserTimezoneResolver`。未設定・不正値時は製品既定 `Asia/Tokyo`。Laravel の `app.timezone=UTC` には落とさない）。
- `/today`・チェックイン・症状観察・プログラム選択の「今日」既定値はユーザー TZ のカレンダー日（`Y-m-d`）。
- `?date=` は任意。不正値はバリデーションエラー（Inertia/ブラウザはリダイレクト＋エラー、JSON は 422）。

## チェックイン API（部分更新）

- `PUT /today/checkin` は **送信されたフィールドのみ更新**する（部分更新）。
- 未送信の `note` / `region_tension` 等は既存値を保持する（null 上書きしない）。

## 実装メモ（2026-07-17 / UI 2026-07-18）

本線（プログラム DAY 生成・チェックイン・作戦カード承認 A・栄養目安）は `/today` に縦通済み。
UI は作戦カード最優先。チェックインは状態バー中心。食事残りは intake（当日合計）を反映。
