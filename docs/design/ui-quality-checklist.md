# UI 品質チェックリスト

UI 仕様書 v2「実装時の品質チェックリスト」を移植したもの。
Clear Dawn の画面（特に TOP ダッシュボード）を実装・レビューする際に全項目を確認する。

## フォント

- [ ] Clear Dawn タイトルが太すぎない
- [ ] タイトルに十分な文字間がある
- [ ] 日本語見出しが上品に見える
- [ ] 本文が読みやすい
- [ ] 手書き風フォントを使いすぎていない

## サイドバー

- [ ] 紫背景が単なる画像っぽくない
- [ ] 月が背景に馴染んでいる
- [ ] 星が主張しすぎていない
- [ ] 植物線画が文字と重なっていない
- [ ] ナビアイコンが SVG で統一されている
- [ ] 下部でも文字が読める

## シート

- [ ] 罫線が強すぎない
- [ ] Excel っぽく見えない
- [ ] 影が重すぎない
- [ ] シート背景が紙っぽい
- [ ] 「今やるべきこと」行が自然に強調されている
- [ ] 中央のシートが主役になっている

## 背景・アイコン

- [ ] 大理石模様が濃すぎない
- [ ] 中央の可読性を邪魔していない
- [ ] 高級ホテル感が強すぎない
- [ ] 白レイヤーで馴染ませている
- [ ] 絵文字を使っていない
- [ ] 線幅が揃っている
- [ ] currentColor で制御できる

## 実装規約（プロジェクトルールとの整合）

- [ ] 色・サイズをマジックナンバーで直書きしていない（トークン経由）
- [ ] 既存の CSS クラスを削除していない
- [ ] 共通 UI コンポーネント（`components/ui/`）の内部を改変していない
- [ ] `<script setup lang="ts">` / `defineProps<T>()` パターンに従っている
